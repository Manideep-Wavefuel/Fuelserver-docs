Communication Policy

Communication Policies are rules or guidelines that helps the server to understand how to communicate with a device. They define things like messaging policies, routing rules, and security protocols. For example, a communication policy might specify how can a server turn off/on or change the color or change the brightness of a "Light Bulb".

This documents helps you to understand and implement Communication Policy. These APIs are specialized tools that help you to perform certain operations on Communication Policy.

With the Help of these APIs you can create, read, update, and delete Communication Policy. The below is the API documentation which contains API url paths, message structure and the description on what the API does. If there is any issue, you can refer to the Errors section to get good insights on the error.

1. Create Communication Policy

This API is used to create a Communication Policy. It takes the below mentioned API parameters as input and returns the newly created Communication Policy and its aws thing type in response.
:organizationId should be replaced with the Id of the Organization.

POST /api/:organizationId/communicationPolicy/create/one → {communicationPolicyAndBaseMessagingPoliciesResult}


Name Type Attribute Description
name string The name of the communication policy. Must be a non-empty string.
protocol string The communication protocol. Must be a non-empty string and one of the valid protocol types.
itemTypeId string The ID of the associated item type. Must be a non-empty string.
isFirmwareControlled boolean | undefined optional (Optional) Indicates whether firmware is controlled. Must be a boolean value (optional).
messageStructure string The message structure of communication policy must be non empty string
confMessageStructure string The confirmation message structure of communication policy must be non empty string
defaultConfMessage string optional (Optional)The default confirmation message structure of communication policy
messageFormat string optional (Optional)The message format of communication policy ID
