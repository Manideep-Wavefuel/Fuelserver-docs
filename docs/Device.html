DEVICE

Devices is the physical object with functionalities or features based on the item it belongs to. Lets say Philips lightbulb is the item, there can be multiple lightbulbs in Philips lightbulb item each lightbulb is a device.

This documents helps you to understand and implement Device. These APIs are specialized tools that help you to perform certain operations on Device.

With the Help of these APIs you can create, read, update, and delete Device. The below is the API documentation which contains API url paths, message structure and the description on what the API does. If there is any issue, you can refer to the Errors section to get good insights on the error.

1. Create Device

This API is used to create a Device. It takes the below mentioned API parameters as input and returns the newly created Device and its aws thing type in response.
:organizationId should be replaced with the Id of the Organization.

POST /api/:organizationId/device/create/one → {CreateDeviceReturn}


Name Type Attribute Description
deviceName string The name of the device. Must be a non-empty string.
itemId string The ID of the associated item. Must be a non-empty string and a valid MongoDB ID.
projectId string The ID of the associated project. Must be a non-empty string and a valid PostgreSQL ID.
serialNumber string The serial number of the device. Must be a non-empty string.
metadata object optional (Optional) The metadata for the device. Must be an object and not empty.
locationId string optional (Optional) The ID of the associated location. Must be a string, not empty, and a valid PostgreSQL ID.

2. Create Many Devices

This API is used to create multiple Devices. It takes the below mentioned API parameters as input and returns the newly created Devices and their AWS thing types in the response.
:organizationId should be replaced with the Id of the Organization.

POST /api/:organizationId/device/create/many → {Array.<CreateDeviceReturn>}


Name Type Attribute Description
data Array.<object> The array of device input object.
data[].deviceName string The name of the device. Must be a non-empty string.
data[].itemId string The ID of the associated item. Must be a non-empty string and a valid MongoDB ID.
data[].projectId string The ID of the associated project. Must be a non-empty string and a valid PostgreSQL ID.
data[].serialNumber string The serial number of the device. Must be a non-empty string.
metadata object optional (Optional) The metadata for the device. Must be an object and not empty.
locationId string optional (Optional) The ID of the associated location. Must be a string, not empty, and a valid PostgreSQL ID.

3. Read Device

This API is used to read a Device. It takes the parameter Id as input in the url and returns the requested Device in response.
:organizationId should be replaced with the Id of the Organization.
:deviceId should be replaced with the Id of the Device.

GET /api/:organizationId/device/read/one/:deviceId → {Device}


4. Read Many Devices

This API is used to read multiple Devices. It takes below parameters as input and return the Devices in response.
:organizationId should be replaced with the Id of the Organization.

POST /api/:organizationId/device/read/many → {Array.<Device>}

5. Update Device

This API is used to update a Device and return the updated Device in the response.
:organizationId should be replaced with the Id of the Organization.
:deviceId should be replaced with the Id of the Device.

PATCH /api/:organizationId/device/update/one/:deviceId → {Device}


Name Type Attribute Description
deviceName string optional (Optional) The name of the device. Must be a non-empty string.
projectId string optional (Optional) The ID of the associated project. Must be a non-empty string and a valid PostgreSQL ID.
metadata object optional (Optional) The metadata for the device. Must be an object and not empty.
locationId string optional (Optional) The ID of the associated location. Must be a string, not empty, and a valid PostgreSQL ID.
provisionState string optional (Optional) The provision state of the device. Must be a string, not empty, and one of the valid provision states (e.g., "PROVISIONED", "DEPROVISIONED", etc.).
deviceStatus string optional (Optional) The status of the device. Must be a string, not empty, and one of the valid device states (e.g., "ACTIVE", "INACTIVE", etc.).

7. Delete Device

:organizationId should be replaced with the Id of the Organization.
:deviceId should be replaced with the Id of the Device.

DELETE /api/:organizationId/device/delete/one/:deviceId → {Device}
