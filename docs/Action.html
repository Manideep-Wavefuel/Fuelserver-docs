Action

Actions are specific tasks or operations that can be performed by a device or group of devices. They are triggered by user input, pre-defined conditions, or communication protocols. An "Action" could be turning on a light based on a motion sensor detection, user sending a message to fan to increase its speed, etc.

This documents helps you to understand and implement Actions. These APIs are specialized tools that help you to perform certain operations on Actions.

With the Help of these APIs you can create, read, update, and delete Actions. The below is the API documentation which contains API url paths, message structure and the description on what the API does. If there is any issue, you can refer to the Errors section to get good insights on the error.

1. Create Action

This API is used to create an Action. It takes the below mentioned API parameters as input and returns the newly created Action in response.
:organizationId should be replaced with the Id of the Organization.

POST /api/:organizationId/action/create/one → {CreateActionReturn}


Name Type Attribute Description
action Array.<object> The array of actions. Must not be empty.
action[].topic string The topic for each action. Must be a non-empty string.
action[].payload object optional The payload for each action. Must be a non-empty object.
action[].options object optional (Optional) The options for each action. Must be an object.
Name Type Attribute Description
qos number | undefined optional (Optional) The QOS value for each action. Must be a number in the range of 0-3.
retain boolean optional (Optional) The Retain value for each action. Must be a boolean.
messageExpiry number optional (Optional) The message expiry for each action. Must be a numeric value representing the number of seconds before expiry.
isUserGenerated boolean The isUserGenerated flag. Must be a boolean.
isAutomated boolean The isAutomated flag. Must be a boolean.
accountId string optional (Optional) The account ID. Must be a string.
actionType boolean optional (Optional) The action type. Must be one of the valid action types.
metadata object optional (Optional) The metadata for the action. Must be an object.

2. Create Many Actions

This API is used to create multiple Actions. It takes the below mentioned API parameters as input and returns the newly created Actions and their vendor in response.
:organizationId should be replaced with the Id of the Organization.

data[].action[].payload object The payload for each action. Must be a non-empty object.
data[].action[].options object optional (Optional) The options for each action. Must be an object.
Name Type Attribute Description

retain boolean optional (Optional) The Retain value for each action. Must be a boolean.
messageExpiry number optional (Optional) The message expiry for each action. Must be a numeric value representing the number of seconds before expiry.
data[].isUserGenerated boolean The isUserGenerated flag. Must be a boolean.
data[].isAutomated boolean The isAutomated flag. Must be a boolean.
data[].accountId string optional (Optional) The account ID. Must be a string.
data[].actionType string optional (Optional) The action type. Must be one of the valid action types.
data[].metadata object optional (Optional) The metadata for the action. Must be an object.

3. Read Action

This API is used to read an Action. It takes the Action Id as input in the url and returns the requested Action in response.
:organizationId should be replaced with the Id of the Organization.
:actionId should be replaced with the Id of the Device.

GET /api/:organizationId/action/read/one/:actionId → {Action}


4. Read Many Actions

7. Delete Action

This API is used to delete an Action and return the deleted Action in response.
:organizationId should be replaced with the Id of the Organization.
:actionId should be replaced with the Id of the Action.

DELETE /api/:organizationId/action/delete/one/:actionId → {Action}


8. Delete Many Actions

This API is used to delete multiple Actions and return the deleted Actions in response.
:organizationId should be replaced with the Id of the Organization.

DELETE /api/:organizationId/action/delete/many → {Array.<Action>}


Name Type Description
ids Array.<string> Array of Action Ids to be deleted.
