ITEM

Items are models of devices within the IoT middleware. They serve as blueprints, defining the expected properties and functionalities of a device like its communication policy, item type, number of components it contains, each component structure, etc. These can be lightbulbs, sensors, smart appliances, and other gadgets that directly interact with the environment and generate/consume data. Example: A device can be a motion sensor, a smart plug, a connected thermostat, or an air purifier.

This documents helps you to understand and implement Item. These APIs are specialized tools that help you to perform certain operations on Item.

With the Help of these APIs you can create, read, update, and delete Item. The below is the API documentation which contains API url paths, message structure and the description on what the API does. If there is any issue, you can refer to the Errors section to get good insights on the error.

1. Create Item

This API is used to create a Item. It takes the below mentioned API parameters as input and returns the newly created Item and its aws thing type in response.
:organizationId should be replaced with the Id of the Organization.

POST /api/:organizationId/item/create/one → {ItemAndItemThingGroupResult}

3. Read Item
