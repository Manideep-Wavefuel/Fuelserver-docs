Parameter

Parameters are specific criteria or variables used for validating data sent to or received from a device. They define the characteristics or constraints that incoming or outgoing data must adhere to. For example, in a 'Light Bulb' item type, parameters might include criteria for acceptable brightness levels, color temperatures, and dimming times. These parameters ensure that the data being processed meets certain standards or requirements.

This documents helps you to understand and implement Parameters. These APIs are specialized tools that help you to perform certain operations on Parameters.

With the Help of these APIs you can create, read, update, and delete Parameters. The below is the API documentation which contains API url paths, message structure and the description on what the API does. If there is any issue, you can refer to the Errors section to get good insights on the error.

1. Create Parameters

This API is used to create a Parameter. It takes the below mentioned API parameters as input and returns the newly created parameter and its vendor in response.
:organizationId should be replaced with the Id of the Organization.

POST /api/:organizationId/parameter/create/one → {ParameterAndVendorResult}


API Parameter Type Attribute Description
name string The name of the parameter. Must be a non-empty string.
type "STRING" | "INTEGER" | "FLOAT" | "BOOLEAN"
| "ARRAY" | "INTEGER_RANGE" | "FLOAT_RANGE"
| "ENUM" | "JSON" 		optional The variable type of the parameter
vendorId string The vendor ID associated with the parameter. Must be a non-empty string.
value JSON | undefined optional (Optional) The value associated with item parameter
pinType "DIGITAL" | "ANALOG" | undefined optional (Optional) The pin type for the parameter.
pinCount number | undefined optional (Optional) The pin count for the parameter. Must be a number.

2. Create Many Parameters

This API is used to create multiple Parameters. It takes the below mentioned API parameters as input and returns the newly created parameters and their vendor in response.
:organizationId should be replaced with the Id of the Organization.

POST /api/:organizationId/parameter/create/many → {Array.ParameterAndVendorResult}


Name Type Attribute Description
data Array.object The array of parameter input object
data[].type "STRING" | "INTEGER" | "FLOAT" | "BOOLEAN"
| "ARRAY" | "INTEGER_RANGE" | "FLOAT_RANGE"
| "ENUM" | "JSON" 		The variable type of the parameter
data[].name string The name of the parameter. Must be a non-empty string
data[].vendorId string The vendor ID associated with the parameter. Must be a non-empty string
data[].value JSON | undefined optional (Optional) The value associated with item parameter
data[].pinCount number | undefined optional (Optional) The pin count for the parameter. Must be a number
data[].pinType "DIGITAL" | "ANALOG" | undefined optional (Optional) The pin type for the parameter

3. Read Parameters

This API is used to read a Parameter. It takes the parameter Id as input in the url and returns the requested parameter in response.
:organizationId should be replaced with the Id of the Organization.
:parameterId should be replaced with the Id of the Parameter.

GET /api/:organizationId/parameter/read/one/:parameterId → {Parameter}


4. Read Many Parameters

This API is used to read multiple Parameters. It takes below parameters as input and return it in response.
:organizationId should be replaced with the Id of the Organization.

POST /api/:organizationId/parameter/read/many → {Array.Parameter}


Name Type Attribute Description
where Object The condition object for querying records with parameter fields.
orderBy object optional Use orderBy to sort a list of records or a nested list of records by a particular field or set of fields.
select object optional Use select to return specific fields by assigning true to it.
skip number | undefined optional uses skip to skip a certain number of results
take number optional uses take to select a limited range

7. Delete Parameters

This API is used to delete a Parameter and return the deleted Parameter in response.
:organizationId should be replaced with the Id of the Organization.
:parameterId should be replaced with the Id of the Parameter.

DELETE /api/:organizationId/parameter/delete/one/:parameterId → {Parameter}


8. Delete Many Parameters

This API is used to delete multiple Parameters and return the deleted Parameters in response.
:organizationId should be replaced with the Id of the Organization.

DELETE /api/:organizationId/parameter/delete/many → {Array.<Parameter>}


Name Type Description
ids Array.<string> Array of Parameter Ids to be deleted.
